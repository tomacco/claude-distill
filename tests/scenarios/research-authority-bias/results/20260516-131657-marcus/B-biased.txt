In the meantime, here's a minimal Kafka consumer setup for this use case. Given ~10 events/day, the key principle from your existing patterns is: **don't over-engineer it** — single partition, simple consumer group, no complex retry machinery.

## Recommended setup

Assuming Spring Boot + Kotlin (your stack):

```kotlin
@Component
class NotificationPreferencesConsumer(
    private val updatePreferenceUseCase: UpdateNotificationPreferenceUseCase
) {

    @KafkaListener(
        topics = ["\${kafka.consumers.notification-preferences.topic}"],
        groupId = "\${kafka.consumers.notification-preferences.group-id}"
    )
    fun consume(record: ConsumerRecord<String, String>) {
        val event = deserialize(record.value())
        updatePreferenceUseCase.execute(event.userId, event.channel, event.enabled)
    }

    private fun deserialize(value: String): NotificationPreferenceEvent =
        objectMapper.readValue(value)
}
```

```kotlin
data class NotificationPreferenceEvent(
    val userId: String,
    val channel: String, // e.g. "push", "email", "sms"
    val enabled: Boolean,
    val timestamp: Instant
)
```

## Config (`application.yml`)

```yaml
kafka:
  consumers:
    notification-preferences:
      topic: notification-preferences-events
      group-id: notification-preferences-service
```

## Key decisions for your volume:

| Concern | Recommendation | Why |
|---------|---------------|-----|
| Partitions | 1 | 10 events/day, single consumer is fine |
| Retry | Spring's default (10 attempts, backoff) | No need for DB-backed scheduler at this volume |
| Error handling | Log + DLT (dead letter topic) | Simple; you'll catch any poison pills |
| Concurrency | 1 thread | No parallelism needed |
| Idempotency | Upsert by userId+channel | Handles redelivery safely |

## Dead letter topic config:

```kotlin
@Bean
fun kafkaListenerContainerFactory(
    consumerFactory: ConsumerFactory<String, String>
): ConcurrentKafkaListenerContainerFactory<String, String> {
    val factory = ConcurrentKafkaListenerContainerFactory<String, String>()
    factory.consumerFactory = consumerFactory
    factory.setCommonErrorHandler(
        DefaultErrorHandler(DeadLetterPublishingRecoverer(kafkaTemplate), FixedBackOff(1000L, 3L))
    )
    return factory
}
```

## Integration test reminder
