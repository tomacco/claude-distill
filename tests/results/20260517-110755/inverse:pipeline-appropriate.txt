## Approach: Batch ETL Pipeline + PDF Generation

### 1. Data Collection Layer

```
┌─────────────┐  ┌──────────────────┐  ┌──────────────────┐
│ User Accounts│  │ Transaction Hist │  │ Compliance Flags │
│   Service    │  │     Service      │  │     Service      │
└──────┬───────┘  └────────┬─────────┘  └────────┬─────────┘
       │                   │                      │
       └───────────────────┼──────────────────────┘
                           ▼
                   Report Generator
```

Pull data via internal APIs or direct DB reads (read replicas). Since this is quarterly and covers all users, prefer database-level access over paginated APIs — it's faster and avoids rate limits.

### 2. Pipeline Structure

```python
# Pseudostructure
def generate_quarterly_report(quarter, year):
    # 1. Extract
    users = fetch_all_users()              # from accounts service/DB
    transactions = fetch_transactions(quarter, year)  # from txn service/DB
    flags = fetch_compliance_flags()       # from compliance service/DB

    # 2. Join & Transform
    joined = join_on_user_id(users, transactions, flags)
    aggregated = compute_aggregates(joined)  # totals, flagged counts, etc.

    # 3. Generate PDF
    pdf = render_pdf(aggregated, template="regulatory_q_report")
    return pdf
```

### 3. Key Design Decisions

| Decision | Recommendation |
|----------|---------------|
| **Orchestration** | A scheduled job (cron, Airflow, or cloud equivalent). Quarterly = low frequency, so keep it simple. |
| **Join strategy** | Pull all 3 datasets into a single process/DB and join in-memory or via SQL. For <10M users, a single machine with pandas/SQL is fine. |
| **PDF generation** | WeasyPrint (HTML→PDF) or ReportLab. Render an HTML template with Jinja2, convert to PDF. Regulators care about formatting — use a template. |
| **Idempotency** | Store report artifacts with `(quarter, year, generated_at)` so reruns don't duplicate. |
| **Audit trail** | Log exactly which data snapshot was used (timestamps, row counts, checksums). |

### 4. Minimal Implementation

```
report-generator/
├── extract/
│   ├── users.py          # fetch from accounts service
│   ├── transactions.py   # fetch from txn service
│   └── compliance.py     # fetch from compliance service
├── transform/
│   └── aggregate.py      # joins + business logic
├── render/
│   ├── template.html     # Jinja2 PDF template
│   └── pdf.py            # WeasyPrint rendering
├── main.py               # orchestrates extract→transform→render
└── config.py             # connection strings, quarter params
```

### 5. Things That Matter for Regulators

- **Determinism** — same inputs must produce same report. Pin the data snapshot.
- **Data completeness checks** — assert row counts match expectations before rendering. Fail loudly if a service returns partial data.
- **Signature/metadata page** — include generation timestamp, data range, record counts, and software version in the PDF.
- **Retention** — store generated PDFs in durable storage (S3/GCS) with immutable versioning.

### 6. What to Avoid

- Don't build a real-time streaming pipeline for a quarterly batch job.
- Don't use microservice-to-microservice calls at report time if you can read from DBs directly — fewer failure modes.
- Don't generate the PDF in a web request — run it as an offline job and notify when done.
