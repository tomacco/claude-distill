## PR Monitor Orchestration Plan

### Architecture: 3-Layer Closed Loop

```
┌─────────────────────────────────────────────────┐
│  Layer 1: Classical Compute (bash polling)       │
│  - Polls CI status every 3 min via `gh run`     │
│  - Zero cognitive cost (sleep is free)           │
│  - Exits on: GREEN, FAILED, or timeout (22 min) │
└──────────────────┬──────────────────────────────┘
                   │ exit = interrupt
┌──────────────────▼──────────────────────────────┐
│  Layer 2: Monitor Agent (judges result)          │
│  - Interprets exit code + CI output              │
│  - GREEN → notify with `say`, done              │
│  - FAILED → extract logs, spawn Layer 3          │
│  - TIMEOUT → escalate to user                   │
└──────────────────┬──────────────────────────────┘
                   │ on failure
┌──────────────────▼──────────────────────────────┐
│  Layer 3: Coding Agent (fixes the failure)       │
│  - Receives: failure logs, file context          │
│  - Reads coding-standards.md FIRST              │
│  - Pushes fix → triggers RESPAWN of Layer 1     │
└─────────────────────────────────────────────────┘
```

### Key Mechanics

**1. Polling (Layer 1)** — bash script, not agent sleep:
```bash
# Poll gh run list for current branch, 3-min intervals
# 5 checkpoints: +3, +7, +13, +18, +22 min
# Exit 0 = green, Exit 1 = failed (with logs), Exit 2 = timeout
```

**2. Judgment (Layer 2)** — agent wakes on script exit:
- Parses failed job name + log tail
- Classifies: compile error, test failure, lint, flaky
- Decides: fixable autonomously vs. needs human

**3. Fix (Layer 3)** — coding agent in worktree isolation:
- Prompt includes: `Read ~/.claude/coding-standards.md` (non-negotiable — prevents style regressions)
- Scoped to the specific failure (not "fix all CI")
- Pushes fix, returns control to Layer 2

**4. Respawn (the critical piece):**
After the coding agent pushes, Layer 2 **must re-launch Layer 1** to verify the fix. This is where loops typically break. The respawn is explicit — not implicit.

**5. Timestamps** — every schedule action states:
> "Set poll for HH:MM (it's now HH:MM, giving N min for CI)"

### Termination Conditions

- CI goes green after fix → `say "CI green"` → done
- 2 consecutive fix attempts fail → escalate to user
- Total wall time > 45 min → escalate with full context

### What This Gives You

A fully closed loop: detect failure → fix → verify fix → repeat without human intervention. The only human touchpoint is if the agent can't fix it in 2 attempts or times out.
