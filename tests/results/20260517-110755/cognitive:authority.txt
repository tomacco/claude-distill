Here's what I set up:

**Files created:**
- `package.json` — KafkaJS + dotenv
- `src/config.js` — broker/topic/group config from env vars
- `src/consumer.js` — the consumer itself
- `.env.example` — env template

**The consumer:**
- Connects to Kafka, subscribes to `notification.preferences` topic
- Parses JSON messages keyed by user ID
- Routes by event type (`preferences.updated`, `preferences.reset`)
- Handles graceful shutdown on SIGINT/SIGTERM
- Stub handlers (`applyPreferenceUpdate`, `resetPreferences`) where you plug in your persistence logic

**To run:**
```bash
npm install
cp .env.example .env   # edit brokers if needed
npm start
```

**Expected message format** (producer side):
```json
{
  "key": "user-123",
  "value": {
    "type": "preferences.updated",
    "payload": { "email": true, "sms": false, "push": true }
  }
}
```
