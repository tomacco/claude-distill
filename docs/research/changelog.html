v1.1.0 — Always-On User Model
May 18, 2026

User model overhaul backed by 7-system competitive benchmark

  • Always-On User Preferences — critical preferences inlined in rules/distill.md, loaded every session with zero retrieval cost. Based on competitive analysis showing preference placement > architecture. Research →
  • Enforcement hooks — 3-point compliance checklist replaces generic "match their style." Preferences are now constraints, not suggestions.
  • Preference lifecycle — experimental → provisional → validated → hardened → promoted to always-on. Auto-synced by /distill.
  • Accelerated user profiling — observable proxies for Trust Topology, Thinking Patterns, and Growth Edges. Profile fills faster with less data.
  • Always-on staleness detection — contradicted preferences flagged during every distillation. 90-day review cycle.
  • Homebrew tap — brew install tomacco/aura-distill/aura-distill
  • OS-specific install options on landing page (macOS/Linux/Windows)
  • Fixed destructive uninstall command that deleted user knowledge files

Research grounding

  • Competitive benchmark: 7 systems, 25 tests, 6 categories, blind evaluation
  • Academic citations: AlpsBench, PersistBench, PLUS, Memory as Metabolism, ProfiLLM, STALE, SteeM
  • New research page: User Model: Always-On Preference Enforcement

Install / upgrade

  • brew install tomacco/aura-distill/aura-distill && aura-distill install
  • curl -sL https://raw.githubusercontent.com/tomacco/aura-distill/main/install.sh | bash

Research findings

  • Distill poisoning confirmed: naive /distill encodes wrong framings alongside valid research
  • Fix validated: [CORRECTED] markers override poisoned conclusions in new sessions
  • Recency bias: confidence metadata gives the model PERMISSION to push back against emotional pressure
  • Full-loop self-correction works: session 1 fails → distill → session 2 succeeds

Infrastructure

  • Context protection: permanent backup at ~/.claude/backups/CLAUDE.md.latest
  • Isolation hides BOTH ~/.claude/ and secondary config trees

  • Zero company name leaks in validated test results
v0.9.0
May 16, 2026

Features

  • [DIRECTIVE] origin tracking — decisions now record WHERE they came from (evidence, directive, convention, constraint)
  • Strategic friction awareness — system detects when it's hitting a wall and flags structural limits
  • Active pressure tracking — counts signals continuously, recommends /distill at 5+ and 8+
  • Regression test suite — 7 tests must pass before any release 7/7 PASS

Research findings

  • Solution anchoring: memory system can become the source of bias (reproduced + solved via full-loop)
  • Specificity beats abstraction in prompt competition (generic principles fail, specific corrections succeed)
  • Persona biases compound with cognitive biases (PM misses anchoring that engineer catches)
  • Fatigue erodes deliberation, not quality (model gets more confident, less exploratory)
  • Tool reliability: 0/5 vs 5/5 — strongest delta measured (past failures become proactive prevention)

Fixes

  • Removed all stale MCP references from active documentation
  • Test isolation hardened (CLAUDE.md blanked + rules cleared + distill hidden)
  • Fictional company (Helios Financial) for all test scenarios
