Research

Empirical findings on LLM knowledge systems. Each study uses A/B testing with identical prompts, models, and conditions. Raw outputs published alongside analysis.

Complete · 5 scenarios
First-Principles Memory vs Flat Files
Same model, same prompt, same session. The only difference: 18 lines of retrieval rules + tiered knowledge files. Does structured memory produce measurably better decisions?
Finding: +6.0 average improvement on 12-point scale. Anti-sycophancy is the killer feature.
Complete · 4 prompts × 2 conditions
Does memory.md Degrade Over Time?
A 200-line memory file (6 months of organic growth) vs distill's tiered structure. Does flat storage rot as knowledge accumulates?
Finding: memory.md WON on one test — revealed a retrieval bug that led to a one-sentence fix.
Complete · 3 scenarios verified
Proportional Surprise: Confidence Scoring
Should all corrections be treated equally? We tested whether assertiveness should scale with accumulated evidence, and what happens when high-confidence beliefs are contradicted.
Finding: all 3 behaviors verified — hardened principles asserted, experimental suggested tentatively, paradigm alarm fired correctly.
Ongoing · 2/5 scenarios
Engineering Principles vs Philosophical Frameworks
Three conditions: engineering-only (root-cause axioms), philosophy-only (Stoic/Pragmatist/Dialectical), and hybrid. Tested on ambiguous decisions with no clear engineering answer.
Early signal: hybrid outperforms both pure approaches. Philosophy reframes; engineering acts.
Ongoing · 4/8 dimensions complete
Cognitive Bias Detection in LLM Interactions
Can structured memory help an LLM detect and surface human cognitive biases — anchoring, loss aversion, decision fatigue, authority — without being paternalistic?
Finding: distill's value varies by bias type. Strong on anchoring/fatigue (+9), subtle on loss aversion (reframing), unique on authority (transparent compliance).

Methodology

Every finding is reproducible. Same model, same prompts, same conditions. Raw outputs published.

Test subject

Claude Code (Opus 4.6) running non-interactively via a second instance ("claudia") with separate config.

A/B framework

Same prompt runs twice: once without knowledge (control), once with distill rules + knowledge files (treatment).

Persona testing

Sofia Chen: fictional lead engineer with realistic cognitive biases, contradictions, and communication style.
